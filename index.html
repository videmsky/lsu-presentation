Hello! Lets make a webmap. Use ↔ to navigate.
First things first, download: Sublime Text 2.
Next, we are going to need access to your computer's shell.
Either use your machine's terminal, or download iTerm2 to make things pretty(assuming you are on a mac).
To begin, we need a basic HTML/CSS/Javascript environment.
We are going to use HTML5 Boilerplate, but there are others.
We will be using Mapbox's API, and need to include their JS and CSS files.
Since Mapbox builds upon Leaflet's API, you should reference their documentation too.
Now for data!
We are going to use USGS's Water Service REST API to fetch instantaneous values.
Use their URL Generation Tool to build a URL that returns JSON.
Here is a sample URL: http://waterservices.usgs.gov/nwis/iv/?format=json&stateCd=la¶meterCd=00060,99133
Now we have all the resources for building our map with USGS data. See this github repo for the relevant code.
Important detail: To get around the browser's same origin policy, we need to run a python server.
To serve up our local files, we need to run the following in the root of our code's folder: python -m SimpleHTTPServer [port]
Replace [port] with: 3000, 8000, etc.
Congratulations! You now have a web map with live data.
Some things to try: Change the parameter codes in the URL to display additional data that the sensor is collecting.
Also, try playing with other tile services. Here are some options.
